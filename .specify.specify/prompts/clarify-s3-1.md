## 一、用户管理（3 个澄清点）

### Q1：用户列表 API 的筛选和分页方式？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §4 + F-ADMIN-USER-001
- **澄清结论**：
  - **筛选**：status（active/pending/suspended/deleted）、role、groupId、keyword（邮箱/姓名模糊搜索）
  - **排序**：createdAt/lastLoginAt/name，支持升序/降序
  - **分页**：offset-based（管理后台列表适合），pageSize 默认 20，最大 100
  - **性能**：1 万用户 P95 ≤ 500ms（含筛选和排序）
- **对 spec 的更新**：在 FR 中定义用户列表 API 参数

### Q2：用户邀请和审批流程？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §4.3 + F-AUTH-008
- **澄清结论**：
  - **邀请**：Admin 输入邮箱 → 系统发送激活链接（有效期默认 7 天，Tenant 可配）→ 用户点击设置密码 → 自动激活（跳过审批）
  - **自注册审批**：用户自行注册 → 状态 pending → Admin 在用户列表 approve/reject
  - **批量邀请**：支持上传邮箱列表（CSV），单次 ≤ 100 人
  - **重复邮箱**：已存在用户跳过 + 返回提示
  - **邀请链接过期**：系统标记为 expired，Admin 可重新发送
- **对 spec 的更新**：在 FR 中定义邀请和审批流程

### Q3：用户删除的完整生命周期？
- **影响级别**：🔴 高
- **文档依据**：PRD_ADMIN §4.3（用户删除）
- **澄清结论**：
  - **即时操作**：设置 status='deleted'，撤销所有 session，移除所有 Group 成员身份
  - **30 天冷静期**：期间 Admin 可恢复用户（重新激活）
  - **30 天后**：PII 字段脱敏（email → hash，name → '已删除用户'），对话/执行记录保留但匿名关联
  - **审计日志**：删除事件及该用户的历史审计日志永久保留（不受 30 天限制）
  - **配额**：删除后不再计入 Group/Tenant 使用量
- **对 spec 的更新**：在 FR 中定义用户删除生命周期

## 二、群组管理（2 个澄清点）

### Q4：Group 成员管理的批量操作？
- **影响级别**：🟢 低
- **文档依据**：PRD_ADMIN §5 + F-ADMIN-GROUP-001
- **澄清结论**：
  - **批量添加**：单次 ≤ 100 人，P95 ≤ 3s；可通过用户选择器或邮箱列表
  - **批量移除**：同上限制
  - **跨 Group 转移**：不直接支持，需先从 A 移除再添加到 B
  - **成员变更即时生效**：权限判定缓存在变更后立即失效
  - **Manager 指派**：一个 Group 可有多个 Manager；用户可同时是多个 Group 的 Manager
- **对 spec 的更新**：在 FR 中定义批量操作 API

### Q5：Group 删除的影响范围？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §5.3
- **澄清结论**：
  - **成员**：全部移至 Unassigned 分组（系统预置，不可删除）
  - **授权**：该 Group 关联的所有 Grant 失效
  - **对话历史**：保留，标记"群组已删除"
  - **配额归因**：删除前的使用数据保留在报表中
  - **下属 Group**：v1.0 不支持分层 Group，因此不存在子 Group 问题
  - **确认机制**：删除前需二次确认（输入 Group 名称）
- **对 spec 的更新**：在 FR 中定义 Group 删除策略

## 三、应用管理（2 个澄清点）

### Q6：应用注册表单的字段和凭证配置？
- **影响级别**：🔴 高
- **文档依据**：PRD_ADMIN §6 + F-ADMIN-APP-001
- **澄清结论**：
  - **基础字段**：name、description、icon（上传或 URL）、tags、category
  - **后端连接**：externalPlatform（dify/coze/n8n）、baseUrl、appId（后端平台的应用 ID）
  - **凭证**：apiKey（AES-256-GCM 加密存储），UI 上保存后不可再查看明文（仅显示 ***）
  - **高级配置**：defaultModel、maxTokens、temperature、systemPrompt
  - **配额**：v1.0 应用级配额不单独配置（用 Tenant/Group/User 三级），字段预留
  - **保存校验**：保存时尝试调用后端 API 验证连接（可选，默认开启）
- **对 spec 的更新**：在 FR 中定义应用注册表单

### Q7：应用启停和删除的行为差异？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §6.4
- **澄清结论**：
  - **禁用（suspend）**：即时生效；已有对话可查看历史，不可发起新对话；应用列表中标灰
  - **删除（delete）**：逻辑删除 status='deleted'；历史对话保留标注"应用已删除"；30 天后清理应用配置数据，对话数据永久保留
  - **恢复**：禁用可随时恢复；删除 30 天内可恢复
  - **凭证处理**：删除后凭证立即清除（不可恢复需重新配置）
- **对 spec 的更新**：在 FR 中定义启停/删除行为

## 四、授权管理（2 个澄清点）

### Q8：授权矩阵的 UI 和操作方式？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §7 + F-ADMIN-AUTHZ-001
- **澄清结论**：
  - **群组→应用授权**：矩阵视图（行=Group，列=App，单元格=Allow/Deny/未配置）
  - **用户直授**：列表视图（userId、appId、permission、expiresAt）
  - **批量操作**：选中多个 Group/App 后一键授权/撤销
  - **权限合并规则**（写死不可配）：Deny > 用户直授 Allow > 群组授权/Manager 用户级授权 Allow > 默认拒绝
  - **生效时间**：授权变更 ≤ 5s 全平台生效（清除权限判定缓存）
  - **审计**：每次授权/撤销操作记录完整审计日志
- **对 spec 的更新**：在 FR 中定义授权管理 UI 和 API

### Q9：用户直授的有效期管理？
- **影响级别**：🟢 低
- **文档依据**：PRD_ADMIN §7.3（直授过期）
- **澄清结论**：
  - **设置**：授权时可选设置 expiresAt（null 表示永久）
  - **自动撤销**：定时任务每分钟扫描过期直授，误差 ≤ 1 分钟
  - **过期通知**：提前 7 天站内信通知 Admin 和用户（可配）
  - **正在使用时过期**：当前进行中的会话可完成，但不可发起新消息
  - **续期**：Admin 可编辑 expiresAt 延长有效期
- **对 spec 的更新**：在 FR 中定义直授有效期管理

## 五、配额与 Manager（2 个澄清点）

### Q10：三级配额的约束关系？
- **影响级别**：🔴 高
- **文档依据**：PRD_ADMIN §8 + F-ADMIN-QUOTA-001
- **澄清结论**：
  - **层级约束**：Tenant 总额 ≥ 所有 Group 配额之和 ≥ 各 User 配额
  - **保存校验**：Group 配额修改时检查是否超过 Tenant 上限；User 配额修改时检查是否超过 Group 上限
  - **超额处理**：如果 Tenant 上限下调导致当前使用量已超标 → 不追溯（既有使用不撤销），但新请求拒绝
  - **配额维度**：Token（输入+输出）、请求次数、成本（货币单位）
  - **重置周期**：默认自然月（Tenant 可配为自然周）
  - **告警阈值**：80%/90%/100% 三级，通过站内信通知对应 Admin
- **对 spec 的更新**：在 FR 中定义配额层级约束和告警规则

### Q11：Manager 权限的边界？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §2.2（Manager 角色定义）
- **澄清结论**：
  - **可见范围**：仅所管理的 Group（可管理多个 Group）
  - **成员管理**：可添加/移除 Group 成员，不可修改用户全局属性
  - **应用授权**：可对 Group 进行应用授权/撤销（不可创建/删除应用）
  - **用户级授权**：可在 Group 范围内给特定用户授权/撤销特定应用
  - **配额**：只读查看 Group 配额使用（不可修改上限）
  - **统计**：查看 Group 级成员排行和应用分布（不可查看 Tenant 级）
  - **不可操作**：用户状态管理、审计日志、安全策略、系统设置
- **对 spec 的更新**：在 FR 中定义 Manager 权限边界
