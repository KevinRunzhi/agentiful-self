S3-2 审计与安全合规闭环

## 切片概述

这是 Phase 1 Stage 3 的第二个切片（S3-2），依赖 S3-1（管理后台核心闭环）。S3-1 已建立完整的管理后台 CRUD 能力。S3-2 的目标是建立审计追溯与安全合规闭环：

1. **审计日志**：5 类事件 100% 采集、不可篡改、按时间/操作者/类型筛选查询、导出
2. **PII 去敏**：敏感字段自动检出（≥ 95%）并按策略脱敏
3. **输出合规**：检测模型输出中的违规内容并按策略拦截
4. **安全策略配置**：认证/SSO/MFA/注入检测策略的管理 UI
5. **统计与报表**：5 维度使用统计、数据延迟 ≤ 5 分钟
6. **成本管理**：成本估算、价格表配置

本切片完成后，平台具备完整的合规审计能力和运营分析能力。

## 覆盖的 Feature（共 11 个）

### 审计与安全

- **F-AUDIT-001** 审计日志采集：5 类事件（认证、授权、数据访问、管理变更、安全事件）100% 覆盖，写入后 ≤ 5s 可查询。
- **F-AUDIT-002** 审计日志不可篡改与权限控制：写入后不可修改/删除（仅系统按保留期自动清理）。保留期 Tenant 可配：180 天～7 年。只有 Tenant Admin 可查询，ROOT ADMIN 需 break-glass。
- **F-AUDIT-003** 高风险访问事由记录：Break-glass、查看他人对话等高风险操作须填写事由，事由附于审计日志不可后期补充。
- **F-SEC-003** PII 去敏配置：自动检测对话/日志/导出中的敏感字段（手机号/邮箱/身份证号/银行卡号等），检出率 ≥ 95%。Tenant Admin 配置脱敏策略（掩码/哈希/移除）。
- **F-SEC-004** 输出内容合规检测与拦截：检测模型输出中的违规内容（暴力/仇恨/成人内容等），Tenant 可配置处置策略（记录/告警/拦截替换）。

### 管理后台 - 安全与审计

- **F-ADMIN-SEC-001** 安全与合规策略配置：认证方式启用/配置、密码策略、MFA 策略、SSO 配置（OIDC/SAML/CAS）、提示词注入检测策略、PII 去敏策略、输出合规策略。策略生效 ≤ 1 分钟。
- **F-ADMIN-AUDIT-001** 审计日志查询与详情查看：按时间范围/操作者/事件类型/对象筛选，查看完整审计字段。30 天查询 P95 ≤ 2s。
- **F-ADMIN-AUDIT-002** 审计日志导出与保留期配置：支持 CSV/JSON 导出，10 万条导出 ≤ 30s。配置保留期（180 天～7 年）。

### 统计与成本

- **F-ADMIN-ANALYTICS-001** 统计与报表：5 维度统计——总览仪表盘、用户使用排行、应用使用统计、模型使用统计、时间趋势分析。数据延迟 ≤ 5 分钟。
- **F-COST-001** 成本估算与价格表配置：基于 Token 使用量和价格表估算成本。按模型/Provider 配置单价。历史成本按历史价格计算不追溯。
- **F-ADMIN-COST-001** 成本估算配置：Admin 管理成本价格表和估算规则。

## User Story 组织建议

- **US1（P0）审计日志查询**：F-AUDIT-001 + F-AUDIT-002 + F-AUDIT-003 + F-ADMIN-AUDIT-001 → Tenant Admin 查询审计日志（按时间/操作者/事件类型/对象筛选），30 天查询 P95 ≤ 2s。查看详情（完整审计字段 + 高风险操作事由）。日志不可篡改。
- **US2（P0）审计日志导出**：F-ADMIN-AUDIT-002 → Tenant Admin 导出审计日志为 CSV/JSON（10 万条 ≤ 30s）。配置保留期（180 天～7 年）。
- **US3（P1）PII 去敏**：F-SEC-003 + F-ADMIN-SEC-001（PII 部分）→ 系统自动检测对话/日志/导出中的 PII 字段（检出率 ≥ 95%）。Admin 配置脱敏策略（掩码/哈希/移除）。
- **US4（P1）输出合规与安全策略**：F-SEC-004 + F-ADMIN-SEC-001（合规 + 安全部分）→ Admin 配置输出合规检测策略、认证/密码/MFA/SSO/注入检测策略。策略变更 ≤ 1 分钟生效。
- **US5（P1）统计与报表**：F-ADMIN-ANALYTICS-001 → Tenant Admin 查看总览仪表盘、用户排行、应用统计、模型统计、时间趋势。数据延迟 ≤ 5 分钟。
- **US6（P1）成本管理**：F-COST-001 + F-ADMIN-COST-001 → Admin 配置价格表（按模型/Provider），查看成本估算（按时间/应用/用户维度）。

## Edge Cases 提示

- **审计日志容量**：高频操作（对话消息）可能产生大量审计事件 → 对话消息级审计只记录会话级别（不逐条消息审计），降低存储压力
- **导出超时**：100 万条导出 → 超出 30s 超时 → 异步导出 + 完成通知下载
- **PII 误检**：正常文本中包含类似手机号的数字串 → 提供 Admin 反馈入口标记误检
- **PII 与审计冲突**：审计日志中包含 PII → 查询时实时脱敏（不修改原始数据），导出时按脱敏策略处理
- **输出合规拦截替换**：模型输出包含违规内容 → 替换为"内容因合规策略被过滤" + 记录完整原文到安全日志（仅 Admin 可查看）
- **统计数据一致性**：实时聚合 vs 预聚合 → 使用预聚合（5 分钟刷新），可能与实时数据有短暂差异
- **价格表变更**：更新价格表后 → 仅影响新的使用量估算，历史成本不追溯
- **保留期到期**：审计日志到保留期后自动清理 → 清理前 30 天通知 Admin → 可延长保留期
