## 一、审计日志（3 个澄清点）

### Q1：5 类审计事件的具体分类？
- **影响级别**：🔴 高
- **文档依据**：PRD §7.1 + F-AUDIT-001
- **澄清结论**：
  - **认证事件**：登录/登出/MFA 验证/密码重置/SSO 绑定/会话过期
  - **授权事件**：授权变更/撤销/直授设置/角色变更/权限判定拒绝
  - **数据访问**：查看对话（含 break-glass）/导出数据/文件下载/审计日志查询
  - **管理变更**：用户增删改/群组增删改/应用增删改/配额调整/策略变更
  - **安全事件**：注入检测/PII 检出/输出合规拦截/异常登录/暴力破解检测
  - **事件字段**：eventId、eventType、eventCategory、actorId、actorRole、targetType、targetId、action、detail（jsonb）、severity、traceId、ipAddress、userAgent、tenantId、createdAt
- **对 spec 的更新**：在 Key Entities 中定义 AuditEvent 实体和 5 类事件枚举

### Q2：审计日志查询的性能优化策略？
- **影响级别**：🟡 中
- **文档依据**：F-ADMIN-AUDIT-001（30 天 P95 ≤ 2s）
- **澄清结论**：
  - **索引策略**：tenantId + createdAt 复合索引；eventCategory + createdAt 复合索引
  - **分区**：按月分区（createdAt range）
  - **查询限制**：默认查询最近 30 天，超过需明确指定时间范围
  - **全文搜索**：v1.0 不实现全文搜索（仅字段精确匹配 + actorId/targetId 筛选）
  - **分页**：cursor-based（基于 eventId + createdAt）
- **对 spec 的更新**：在 NFR 中定义查询性能约束

### Q3：审计日志导出的流式处理？
- **影响级别**：🟡 中
- **文档依据**：F-ADMIN-AUDIT-002（10 万条 ≤ 30s）
- **澄清结论**：
  - **同步导出**：≤ 10 万条直接流式下载（CSV/JSON），P95 ≤ 30s
  - **异步导出**：> 10 万条转为异步任务 → 生成文件 → 站内信通知 → 24 小时内通过链接下载
  - **PII 处理**：导出时按 Tenant 配置的脱敏策略处理敏感字段
  - **导出事件**：导出操作本身记录审计日志（谁导出了什么时间范围的日志）
  - **格式**：CSV（默认），JSON（可选）
- **对 spec 的更新**：在 FR 中定义导出 API 和异步任务流

## 二、PII 与输出合规（3 个澄清点）

### Q4：PII 检测的规则引擎？
- **影响级别**：🟡 中
- **文档依据**：PRD §8.2 + F-SEC-003
- **澄清结论**：
  - **检测方式**：正则表达式 + 模式匹配（v1.0 不使用 AI 分类器）
  - **检测范围**：手机号（中国/国际）、邮箱地址、身份证号（中国）、银行卡号、信用卡号
  - **检出率**：≥ 95%（基于标准格式，非标格式可能遗漏）
  - **检测时机**：
    - 审计日志属于、导出时：实时检测 + 脱敏
    - 对话内容：仅在导出/分享时脱敏，对话中原文展示（不影响用户体验）
  - **脱敏策略**（Tenant 可配）：
    - `mask`（默认）：部分遮挡（如 138****1234）
    - `hash`：SHA-256 哈希替换
    - `remove`：完全移除
- **对 spec 的更新**：在 FR 中定义 PII 检测引擎和脱敏策略

### Q5：输出合规检测的规则和处置？
- **影响级别**：🟡 中
- **文档依据**：PRD §8.3 + F-SEC-004
- **澄清结论**：
  - **检测方式**：关键词匹配 + 正则模式（v1.0 不使用 AI 分类器）
  - **检测类别**：暴力内容、仇恨言论、成人内容、政治敏感（中国区）、自伤倾向
  - **检测时机**：SSE 流式输出完成后（不中断流式，事后检测）
  - **处置策略**（Tenant 可配）：
    - `log`（默认）：仅记录安全日志
    - `alert`：记录 + 站内信通知 Admin
    - `block`：替换消息内容为"内容因合规策略被过滤"，原文存入安全日志（仅 Admin 可查看）
  - **用户体验**：block 模式下消息已展示后再替换 → 前端展示简短过渡动画 + 合规提示
- **对 spec 的更新**：在 FR 中定义输出合规检测管线

### Q6：安全策略配置的 UI 结构？
- **影响级别**：🟢 低
- **文档依据**：PRD_ADMIN §10 + F-ADMIN-SEC-001
- **澄清结论**：
  - **页面结构**：单页面，分 Tab 展示
    - 认证策略：启用的认证方式（密码/Google/GitHub/SSO 等）
    - 密码策略：最小长度/复杂度/有效期/历史限制
    - MFA 策略：强制/可选/关闭
    - SSO 配置：OIDC/SAML/CAS 连接参数
    - 注入检测：策略（log/alert/block）+ 自定义关键词
    - PII 去敏：策略（mask/hash/remove）+ 敏感字段类型开关
    - 输出合规：策略（log/alert/block）+ 检测类别开关
  - **生效时间**：保存后 ≤ 1 分钟全平台生效（通过配置版本号机制）
  - **修改审计**：每次策略变更记录完整的 before/after diff
- **对 spec 的更新**：在 FR 中定义安全策略配置页面

## 三、统计与成本（3 个澄清点）

### Q7：统计报表的 5 个维度？
- **影响级别**：🟡 中
- **文档依据**：PRD_ADMIN §9 + F-ADMIN-ANALYTICS-001
- **澄清结论**：
  - **总览仪表盘**：总用户数、活跃用户数（DAU/MAU）、总对话数、总 Run 数、总 Token 消耗、总成本
  - **用户使用排行**：按 Token/请求次数/成本排序的用户列表 TOP 20
  - **应用使用统计**：按应用维度的使用量分布（对话数/Token/成本）
  - **模型使用统计**：按模型维度的使用量分布（Token/成本/请求数）
  - **时间趋势分析**：选择时间范围（7 天/30 天/90 天）的使用量折线图
  - **数据来源**：预聚合表，每 5 分钟刷新
  - **Manager 范围**：Manager 只能查看所管理 Group 的统计数据
- **对 spec 的更新**：在 FR 中定义 5 个统计维度和 API

### Q8：成本估算的计算逻辑？
- **影响级别**：🟢 低
- **文档依据**：F-COST-001 + F-ADMIN-COST-001
- **澄清结论**：
  - **价格表**：按 model + provider 配置单价（输入 Token 单价/输出 Token 单价）
  - **默认值**：系统预置常见模型价格（如 GPT-4o、Claude 3.5 等），Admin 可覆盖
  - **计算**：cost = inputTokens × inputPrice + outputTokens × outputPrice
  - **聚合维度**：按用户/应用/模型/时间聚合
  - **历史处理**：价格表变更后不追溯历史，历史成本保持原价计算
  - **货币**：v1.0 仅支持 USD，汇率转换在 v2.0+
- **对 spec 的更新**：在 FR 中定义价格表和成本计算逻辑

### Q9：预聚合表的刷新策略？
- **影响级别**：🟢 低
- **文档依据**：无直接文档依据
- **澄清结论**：
  - **刷新频率**：每 5 分钟运行一次聚合任务
  - **聚合粒度**：按小时聚合存储，查询时按天/周/月动态计算
  - **聚合表**：analytics_hourly（tenantId、hour、dimension、dimensionValue、tokenCount、requestCount、cost）
  - **延迟容忍**：仪表盘数据延迟 ≤ 5 分钟
  - **历史数据**：保留所有历史聚合数据（不设清理期限，但可按需归档到冷存储）
- **对 spec 的更新**：在 Key Entities 中定义 analytics_hourly 表
