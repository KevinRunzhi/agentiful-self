# AgentifUI 非功能约束 (NFR Baseline)

* **规范版本**：v0.2
* **最后更新**：2026-01-26
* **状态**：草稿，待评审
* **参考**：PRD 性能指标 + OpenTelemetry 规范 + LobeChat

---

## 1. 概述

本文档定义 AgentifUI 的非功能需求（NFR）基线，作为系统设计和验收的约束条件。

### 适用范围

- 统一接口网关
- 核心服务层
- 前端应用

---

## 2. 性能约束

### 2.1 响应时间

| 指标 | 目标值 | 测量方式 | 来源 |
|------|--------|----------|------|
| 对话首 Token 时间 | P95 ≤ 1.5s | 用户发送到首字符显示 | PRD 6.9 |
| 权限判定 | P95 ≤ 50ms | 网关内部处理时间 | PRD 4.3 |
| SSO 匹配 | P95 ≤ 500ms | 邮箱输入到结果返回 | PRD 3.4 |
| 停止响应 | ≤ 500ms | 点击到停止渲染 | PRD 6.9 |
| 状态更新 | ≤ 3s | 状态变更到 UI 反映 | PRD 7.5 |
| 审计可查 | ≤ 5s | 事件发生到可查询 | PRD 8.6 |

### 2.2 吞吐量

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 网关 QPS | ≥ 1000 | 单实例，常规请求 |
| 流式连接 | ≥ 500 | 单实例，并发 SSE 连接 |
| 消息处理 | ≥ 10,000/min | 消息持久化吞吐 |

### 2.3 并发

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 在线用户 | ≥ 10,000 | 单实例支持 |
| 并发会话 | ≥ 1,000 | 活跃对话会话 |

## 6. 可维护性约束

### 6.1 可观测性

| 能力 | 要求 |
|------|------|
| 日志 | 结构化 JSON (Pino) |
| 追踪 | OpenTelemetry，100% 覆盖 |
| 指标 | Prometheus 格式导出 |
| 告警 | 关键指标告警 |

**Span 属性命名规范**（参考 OpenTelemetry 语义约定）：

| 类别 | 属性名 | 说明 |
|------|--------|------|
| 服务器 | `server.address` | 主机名 |
| 请求 | `http.request.method` | HTTP 方法 |
| 请求 | `url.full` | 完整 URL |
| 请求 | `url.path` | 路径 |
| 请求 | `http.route` | 路由模板 |
| 响应 | `http.response.status_code` | 状态码 |
| 错误 | `error.name` | 错误名称 |
| 错误 | `error.message` | 错误消息 |
| 错误 | `error.stack` | 错误堆栈 |

**请求 ID 规范**：

| 项目 | 规范 | 说明 |
|------|------|------|
| 格式 | UUID v7 | 时间有序，兼顾唯一性和排序性 |
| 生成位置 | 网关 `genReqId()` | Fastify 实例创建时配置 |
| 响应头 | `X-Request-ID` | 返回给客户端供排查 |

**Trace ID 规范**：

| 项目 | 规范 | 说明 |
|------|------|------|
| 格式 | W3C Trace Context | 32 位十六进制字符串 |
| 传播 | `traceparent` 请求头 | 跨服务传播 |
| 响应头 | `X-Trace-ID` | 返回给客户端供观测平台跳转 |

### 6.2 部署

| 能力 | 要求 |
|------|------|
| 容器化 | Docker 镜像 |
| 编排 | Docker Compose / K8s |
| 零停机部署 | 滚动更新 |
| 回滚 | ≤ 5min |

### 6.3 配置

| 能力 | 要求 |
|------|------|
| 配置方式 | 环境变量 + 配置文件 |
| 热更新 | 非敏感配置支持 |
