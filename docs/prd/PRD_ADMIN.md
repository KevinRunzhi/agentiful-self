## 1. 概述

### 1.1 背景与目标

AgentifUI 作为企业级 AI 应用统一管理平台，需要为不同角色的管理者提供对应的后台管理能力。管理后台是实现平台治理、安全合规、资源配置的核心入口。

**目标**：
- 为 ROOT ADMIN / Tenant Admin / Manager 提供与其权限匹配的管理能力
- 实现用户、群组、应用、权限、配额的全生命周期管理
- 提供可视化的统计分析与审计追溯能力
- 支持安全合规策略的配置与监控

### 1.2 目标用户与权限边界

- **ROOT ADMIN**
  - 可见范围：全平台（仅启用时）
  - 核心职责：租户生命周期、全局配置

- **Tenant Admin**
  - 可见范围：当前 Tenant
  - 核心职责：用户/群组/应用/权限/配额/策略/审计

- **Manager**
  - 可见范围：所管理 Group
  - 核心职责：成员管理、应用授权、配额查看、团队统计

### 1.3 设计原则

- **权限隔离**：严格按角色裁剪可见功能与数据范围
- **最小权限**：仅展示角色职责所需的管理能力
- **操作可审计**：所有管理操作记录审计日志
- **自助优先**：用户自助能力优先，减少管理员介入
- **PC 端专属**：管理后台仅支持 PC 端访问（Min-width: 1024px），不承诺移动端适配

## 3. 平台管理（ROOT ADMIN）

> 仅当 ROOT ADMIN 显式启用时可见

### 3.1 租户管理

**背景**：ROOT ADMIN 负责多租户环境下的租户生命周期管理。

**功能范围**：

**In-Scope**：
- **租户列表**：查看所有租户及状态
- **创建租户**：创建新 Tenant，配置基础信息
- **编辑租户**：修改租户名称、描述、配置
- **禁用/启用租户**：暂停或恢复租户访问
- **删除租户**：【高危】删除租户及业务数据，审计日志按法务保留期延迟清理
- **租户配额总额**：配置租户级 Token/成本上限

**Out-of-Scope**：
- **查看租户业务内容**：查看用户对话等业务数据（默认禁止）

**紧急访问（Break-glass）**：

> 仅限运维排障场景下使用，属于受限 In-Scope。

操作流程：
1. 填写强制访问事由（必填）
2. 系统生成最高级别审计日志（Severity=Critical），不可删除
3. 系统自动向 Tenant Admin 发送**站内信**通知（邮件通知为 v2.0+）
4. 访问权限仅在本次会话或 1 小时内有效

**核心规则**：
- 删除租户需二次确认 + 强制审计
- ROOT ADMIN 默认不查看业务内容，除非显式 break-glass 授权

**验收标准**：
- 租户创建后 ≤ 30s 可正常使用
- 租户禁用后即时生效，所有会话强制登出

### 3.2 全局配置

- **默认认证方式**：新租户默认启用的认证方式
- **默认安全策略**：新租户默认的安全策略模板
- **外部观测平台**：全局默认 Trace 跳转配置
- **平台维护公告**：全平台维护通知

## 4. 用户管理（Tenant Admin）

### 4.1 背景与目标

管理 Tenant 内所有用户的生命周期，包括审批、状态管理、资料维护。

### 4.2 功能范围

- **用户列表**：搜索、筛选、分页浏览用户（Tenant Admin）
- **用户详情**：查看用户资料、群组归属、权限、配额使用（Tenant Admin）
- **邀请用户**：通过邮箱邀请用户加入 Tenant（Tenant Admin），发送激活链接，用户点击后设置密码并自动激活（跳过审批）；链接有效期默认 7 天
- **用户审批**：审批待审核用户（通过/拒绝）（Tenant Admin）
- **状态管理**：激活/暂停/解锁/重新审批用户（Tenant Admin）
- **重置密码**：发送密码重置链接（Tenant Admin）
- **强制 MFA**：为指定用户强制启用 MFA（Tenant Admin，优先级高于 Tenant 级策略）
- **编辑资料**：修改用户昵称、邮箱（受限）（Tenant Admin）
- **删除用户**：【高危】删除用户及关联数据（Tenant Admin）
- **查看成员对话**：【受限】查看指定用户对话内容（Tenant Admin，需填写事由，自动生成审计日志）

### 4.3 核心规则

**用户状态流转**：

- **待审核**
  - 通过审批 → 转为 **活跃**
  - 拒绝审批 → 转为 **已拒绝**

- **已拒绝**
  - 重新审批通过 → 转为 **活跃**（需管理员主动发起）

- **活跃**
  - 执行暂停操作 → 转为 **已暂停**

- **已暂停**
  - 执行激活操作 → 转为 **活跃**

**待审核用户**：
- 可登录，仅可访问个人设置页与状态提示
- 不可访问应用、对话、群组功能
- **默认群组**：无默认群组（Unassigned），需管理员邀请或分配至具体 Group 方可使用业务功能

**用户删除**：
- 软删除，保留审计关联
- 对话/执行记录保留（脱敏处理）
- 30 天后可彻底清理

### 4.4 验收标准

- **用户列表加载**：P95 ≤ 500ms（1万用户）
- **审批操作**：即时生效
- **状态变更**：≤ 5s 全平台生效

## 5. 群组管理（Tenant Admin / Manager）

### 5.1 背景与目标

管理 Tenant 内的 Group 组织结构，支持成员管理与授权配置。

### 5.2 功能范围

**Tenant Admin 权限**：
- 群组列表（全部群组）
- 创建群组
- 编辑群组
- 删除群组
- 添加成员
- 移除成员
- 指派 Manager
- 查看群组统计

**Manager 权限**：
- 群组列表（仅所管理群组）
- 编辑群组（仅所管理）
- 添加成员（仅所管理）
- 移除成员（仅所管理）
- 查看群组统计（仅所管理）
- 用户级应用授权（仅所管理，限于 Tenant Admin 已授权给该 Group 的应用范围内，为具体用户分配授权；可选填写授权原因，纳入审计）

### 5.3 核心规则

**群组创建**：
- 必填：群组名称
- 可选：描述、默认配额策略

**成员管理**：
- 支持批量添加（邮箱列表）
- 移除成员不删除用户，仅解除群组关系
- 用户移除后即时失去该群组授权的应用访问权

**Manager 指派**：
- 同一用户可担任多个 Group 的 Manager
- Manager 权限仅在所管理 Group 内生效

**群组删除**：
- 【高危】需二次确认 + 强制审计
- 成员归属：移至 Unassigned（无群组），待管理员重新分配
- 历史数据：群组上下文产生的对话/执行记录只读保留，标记"群组已删除"
- 配额归因：删除前的使用数据保留于报表

### 5.4 验收标准

- **成员变更**：权限即时生效
- **批量添加**：单次 ≤ 100 人，P95 ≤ 3s

## 6. 应用管理（Tenant Admin）

### 6.1 背景与目标

管理 Tenant 内已注册的 AI 应用，配置应用信息与后端连接。

### 6.2 功能范围

- **应用列表**：查看所有已注册应用
- **注册应用**：新增应用，配置后端连接
- **编辑应用**：修改应用信息与配置
- **禁用/启用应用**：临时下线或恢复应用
- **删除应用**：【高危】删除应用定义
- **应用授权管理**：配置应用可见的群组
- **应用策略配置**：配置应用级文件上传/合规策略（仅可在 Tenant 级限制内收紧，不可放宽）

### 6.3 应用注册配置项

**必填项**：
- `name`：应用名称，用户可见名称
- `type`：应用类型（Chatbot / Chatflow / Agent）
- `engine`：后端引擎（Dify / Coze / n8n / 自定义）
- `endpoint`：后端引擎上游地址
- `app_id`：后端应用标识
- `credentials`：API Key 或集成凭证

**可选项**：
- `icon`：应用图标
- `description`：功能说明
- `welcome_message`：对话初始提示语
- `tags`：用途标签（如"文案创作"、"数据分析"），用于应用发现分类浏览

### 6.4 核心规则

- 应用禁用后，已有对话可查看历史，不可发起新对话
- 应用删除后，历史对话保留但标记为"应用已删除"
- 凭证以加密方式存储，仅网关层可解密使用

### 6.5 验收标准

- **应用注册**：保存后 ≤ 10s 可使用
- **应用禁用**：即时生效

## 7. 权限与授权（Tenant Admin）

### 7.1 背景与目标

配置用户、群组与应用之间的授权关系，管理直授例外。

### 7.2 功能范围

- **群组-应用授权**：配置群组可访问的应用列表
- **用户-应用直授**：【例外】为用户直接授权应用
- **直授记录查看**：查看所有用户直授及有效期
- **批量授权**：一次性为多个群组授权应用
- **授权撤销**：撤销群组或用户的应用授权

### 7.3 核心规则

**群组授权（主路径）**：
- 授权后，群组所有成员即时获得应用访问权
- 撤销后即时失效

**用户直授（例外路径，仅 Tenant Admin）**：
- **必须填写**：授权原因、有效期
- **有效期限制**：默认 7 天，最长 90 天
- 到期自动撤销，支持手动提前撤销
- 纳入审计，支持导出直授报表

**Manager 用户级授权（受限，与用户直授不同）**：
- 依托群组授权范围，无有效期限制（跟随群组授权生命周期）
- 可选填写授权原因
- 纳入审计

**权限合并规则**（写死）：
- 访问权限：**并集（OR）**
- 配额归因：**当前工作群组**

### 7.4 验收标准

- **授权生效**：≤ 5s
- **直授过期**：准时自动撤销（误差 ≤ 1 分钟）

## 8. 配额管理（Tenant Admin / Manager）

### 8.1 背景与目标

配置多层级配额限制，监控配额使用与告警。

### 8.2 功能范围

**Tenant Admin 权限**：
- Tenant 配额配置
- Group 配额配置
- User 配额配置
- 配额使用查看（全部）
- 告警阈值配置
- 告警通知配置

**Manager 权限**：
- 配额使用查看（仅所管理群组）

### 8.3 配额配置项

**Tenant 级**：
- `token_limit_monthly`：Token 总额（月），租户级硬上限
- `cost_limit_monthly`：成本总额（月），租户级成本上限

**Group 级**：
- `token_quota_monthly`：Token 配额（月），群组级分配额度
- `cost_quota_monthly`：成本配额（月），群组级成本上限

**User 级**：
- `token_quota_monthly`：Token 配额（月），个人使用上限

### 8.4 告警配置

- **告警阈值**：默认 80%、90%、100%
- **通知方式**：站内通知（必选）、Webhook（可选）
- **通知对象**：用户本人 + Manager + Tenant Admin

### 8.5 核心规则

- **优先级**：细粒度约束优先，不允许突破 Tenant 总上限
- **配额重置周期**：自然月（可配置为自然周）
- **超配额后**：拒绝新请求，已有请求完成执行

### 8.6 验收标准

- **配额检查**：每请求 ≤ 50ms
- **告警延迟**：触达阈值后 ≤ 5 分钟发出

## 9. 统计与报表（Tenant Admin / Manager）

### 9.1 背景与目标

提供多维度使用统计与成本分析，支持资源规划与决策。

### 9.2 功能范围

**Tenant Admin 权限**：
- 总览仪表盘（Tenant 级）
- 用户使用排行（全部）
- 应用使用统计
- 模型使用统计
- 成本分析
- 趋势分析
- 报表导出

**Manager 权限**：
- 总览仪表盘（Group 级）
- 用户使用排行（仅所管理群组）
- 应用使用统计（仅已授权应用）
- 成本分析（仅所管理群组）
- 趋势分析（仅所管理群组）
- 报表导出（仅所管理群组）

**Manager 数据可见性特别说明**：
- Manager 可见成员在 **当前 Group 上下文** 中产生的使用数据、成本与用户级指标排行（仅指标，不含对话内容）
- 严禁聚合展示用户在平台其他 Group 或个人空间的行为数据，保障隐私边界

### 9.3 统计维度

**时间维度**：今日 / 本周 / 本月 / 自定义区间

**主体维度**：Tenant / Group / User / App / Model

**指标维度**：请求量、成功率、Token、成本、P95 延迟

### 9.4 报表格式

- 在线图表：折线图、柱状图、饼图
- 导出格式：CSV、Excel

### 9.5 验收标准

- **仪表盘加载**：P95 ≤ 2s
- **数据延迟**：≤ 5 分钟（准实时）

## 10. 安全与合规（Tenant Admin）

### 10.1 背景与目标

配置 Tenant 级安全策略与合规规则，监控安全事件。

### 10.2 功能范围

- **认证配置**：启用/配置各认证方式
- **密码策略**：配置密码复杂度要求
- **MFA 策略**：配置 MFA 强制/可选/关闭
- **SSO 配置**：配置 OIDC/SAML/CAS
- **提示词注入检测**：配置检测策略（记录/告警/拦截）
- **PII 去敏配置**：配置去敏范围与规则（v1.0 仅支持不可还原去敏，关联追溯功能为 v2.0+）
- **内容合规策略**：配置输出拦截规则
- **安全事件监控**：查看安全告警与处理

### 10.3 认证配置项

- **邮箱密码**：启用/关闭
- **手机号**：启用/关闭、第三方服务配置
- **Google/GitHub/WeChat**：启用/关闭、OAuth 配置
- **OIDC**：Issuer URL、Client ID/Secret、Scope
- **SAML 2.0**：IdP Metadata、证书
- **CAS**：CAS Server URL

### 10.4 密码策略配置

- **最小长度**：默认 8，可配范围 8-32
- **复杂度要求**：默认大小写+数字，可增加特殊字符
- **密码有效期**：默认不过期，可配 30-365 天
- **历史密码限制**：默认不限制，可配最近 3-12 个

### 10.5 安全事件类型

- **异常登录**：默认告警，可配拦截/二次验证
- **提示词注入**：默认记录+告警，可配拦截
- **内容违规**：默认拦截，可配记录+告警
- **高频调用**：默认告警，可配限流/拦截

### 10.6 验收标准

- **策略生效**：≤ 1 分钟
- **安全事件展示**：≤ 1 分钟延迟

## 11. 审计日志（Tenant Admin）

### 11.1 背景与目标

提供完整的操作审计记录，支持合规检查与问题追溯。

### 11.2 功能范围

- **日志查询**：按时间、操作者、事件类型、对象筛选
- **日志详情**：查看完整审计字段
- **日志导出**：导出查询结果（CSV/JSON）
- **保留期配置**：配置日志保留时长

### 11.3 审计事件类别

- **鉴权事件**：登录/登出、失败登录、Token 操作
- **权限变更**：角色变更、授权变更、成员变更
- **资源操作**：创建/编辑/删除 用户/群组/应用
- **配额变更**：配额配置修改、阈值调整
- **安全事件**：拦截、脱敏、告警触发
- **高风险访问**：查看他人对话、导出数据

### 11.4 审计字段

审计日志包含操作者、时间、对象、动作、结果、来源 IP/UA、事由、Trace ID 等通用字段。

> 详细字段定义参见附录 B《审计事件模型规范》。

### 11.5 核心规则

- **不可篡改**：日志不可编辑、不可删除
- **保留期**：默认 180 天，可配最长 7 年
- **导出权限**：仅 Tenant Admin，需审计记录

### 11.6 验收标准

- **日志写入**：事件发生后 ≤ 5s 可查询
- **查询性能**：30 天内日志查询 P95 ≤ 2s
- **导出性能**：10 万条记录导出 ≤ 30s

## 12. 系统设置（Tenant Admin）

### 12.1 Tenant 基础信息

- `name`：Tenant 名称（组织名称）
- `description`：Tenant 描述（组织描述）
- `contact_email`：管理员联系邮箱

### 12.2 主题与品牌

- `primary_color`：品牌主色调
- `secondary_color`：辅助色
- `logo`：头部 Logo 图片
- `favicon`：浏览器图标
- `site_name`：浏览器标题
- `login_page`：登录页 Logo、背景、欢迎语

### 12.3 多语言

- `default_language`：新用户默认语言（中文/英文）
- `allow_user_override`：是否允许用户自选语言

### 12.4 文件上传策略（Tenant 级默认值）

- `max_file_size`：单文件大小上限，默认 50 MB，可配 ≤ 200 MB
- `allowed_file_types`：允许的文件类型白名单
- `max_files_per_upload`：单次上传数量上限，默认 10 个，可配 ≤ 20 个
- `file_retention_days`：文件保留期，默认 90 天，可配 30 天 ~ 永久

> 应用级策略可在此基础上收紧，不可放宽。

### 12.5 对话分享配置

- `share_default_expiry_days`：分享链接默认有效期（天），默认永久（0 表示永久，不受最长有效期限制）
- `share_max_expiry_days`：分享链接最长有效期（天），默认 365 天，仅对非永久分享生效
- `share_require_login`：是否强制登录查看，默认启用；启用时仅同 Tenant/同群组可见，关闭时为公开链接

### 12.6 Webhook 配置

- `webhook_url`：接收端点
- `subscribed_events`：选择推送的事件类型
- `signing_secret`：签名校验密钥
- `enabled`：启用/禁用
- `delivery_logs`：查看投递记录与状态，支持失败事件重试/重放

### 12.7 外部观测平台

- `platform_type`：平台类型（Grafana / Jaeger / 自定义）
- `url_template`：包含 `{trace_id}` 占位符的跳转 URL
- `enabled`：启用/禁用

### 12.8 通知配置

- `in_app_notifications`：站内通知（配额告警、审批结果、系统公告）
- `notification_retention`：通知保留期，默认 90 天

### 12.8a 系统公告管理

**功能范围**：
- **创建公告**：编写标题/正文，选择展示方式（Banner/模态弹窗）
- **发布公告**：即时发布或定时发布
- **结束公告**：手动撤下或设定自动过期时间
- **公告历史**：查看已发布/已过期公告

**展示方式**：
- **顶部 Banner**：全员可见，可关闭
- **模态弹窗**：紧急通知，首次登录强制展示

### 12.9 API Key 管理

- `api_keys`：API Key 列表（创建/撤销/查看使用统计）
- `key_name`：API Key 名称（必填）
- `scopes`：授权范围（只读/读写/特定模块）
- `expires_at`：过期时间（可选，默认永久）
- `last_used_at`：最近使用时间（只读）

**核心规则**：
- 单 Tenant 最多 20 个 API Key
- Key 仅创建时展示一次，不可再次查看
- 撤销后立即失效

### 12.10 保留策略汇总

> 统一视图便于合规人员一站式审查。

| 数据类型 | 默认保留期 | 可配范围 | 配置入口 |
|---------|----------|---------|----------|
| 上传文件 | 90 天 | 30 天 ~ 永久 | 12.4 |
| 站内通知 | 90 天 | - | 12.8 |
| 审计日志 | 180 天 | 最长 7 年 | 11.5 |

## 13. 成本估算配置（Tenant Admin）

### 13.1 背景与目标

配置模型价格以支撑成本估算与报表。

### 13.2 功能范围

- **价格表管理**：配置各模型的 Token 单价
- **币种配置**：选择成本展示币种（USD/RMB）
- **税费配置**：配置含税/不含税视图

### 13.3 价格表配置

- `model_name`：模型标识
- `input_token_price`：每千 Token 输入价格
- `output_token_price`：每千 Token 输出价格
- `effective_date`：价格生效起始时间

### 13.4 核心规则

- 默认使用平台内置价格表
- Tenant 可覆盖以匹配企业实际结算价
- 历史成本按历史价格计算，不追溯

## 14. 验收标准汇总

### 14.1 性能指标

- **用户列表（1万用户）**：P95 ≤ 500ms
- **群组列表（100群组）**：P95 ≤ 300ms
- **应用列表（50应用）**：P95 ≤ 300ms
- **审计日志查询（30天）**：P95 ≤ 2s
- **统计仪表盘**：P95 ≤ 2s
- **配置保存**：P95 ≤ 1s

> 注：复杂列表查询允许 ≤ 800ms，简单交互应均满足 ≤ 300ms。

### 14.2 功能指标

- **权限变更生效**：≤ 5s
- **状态变更生效**：≤ 5s
- **配额检查**：≤ 50ms
- **审计日志写入**：≤ 5s
- **告警发出**：≤ 5 分钟

### 14.3 安全指标

- **所有操作可审计**：100%
- **敏感操作二次确认**：100%
- **权限隔离**：0 越权访问
